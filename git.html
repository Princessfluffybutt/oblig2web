Git og GitHub

The Git in GitHub. Git er et Source Code Management system (SCM) som er utviklet av Linus Torvalds i 2005. Grit er et verktøy for versjonskontroll og kodedeling. Det kjennetegnes ved at det er distribuert og ikke mappehierarki-basert. Det brukes til å redigere filer fra en mappe på en server, et oppbevaringssted eller repository (repo) samtidig.

Git holder kontroll på alt av innhold gjennom en trestruktur og commit-referanser. Du kan lagre utviklings-historikken til en kode, og gå tilbake til enhver endring (commit). Strukturen gjør at man oppnår en vesentlig høyere hastighet en de klassiske SCM-verktøyene. Når man jobber på store programmeringsprosjekter sammen kan dette være et godt verktøy, og alternativ til for eksempel Dropbox/Google Drive Man kan lage flere versjoner av et program for publisering og Grit kan løse mange konflikter som kommer av forskjellig kode på ulike maskiner. Dette gjør det lettere å samarbeide om den samme koden.

Git er en type programvare og protokoll. GitHub er en av flere tjenester som du lagrer kode i. Andre tjenester er for eksempel BitBucket eller GitLab. Med et slikt verktøy kan man få eksponering og tilbakemelding fra programmeringsmiljøet på prosjekter. Man har oversikt over hva som har blitt forandret i koden hver gang i tillegg til at den er kompatibel men plattformer som Amazon, Google Cloud og Code Climate.
